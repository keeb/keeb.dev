MCP Is Dead

When Anthropic shipped the Model Context Protocol in late 2024, the developer community treated it like the second coming of REST. A universal, open standard for connecting AI models to external tools, data sources, and services — what wasn't to love? Early adopters spun up MCP servers for everything from Postgres introspection to Kubernetes cluster management. The spec was clean, the reference implementations were solid, and for about six months it genuinely felt like the ecosystem was converging on a single interoperability layer. Conference talks were titled things like "MCP: The Last Protocol You'll Ever Need." We should have known better.

The fracture started quietly. Google shipped Agent2Agent (A2A) with a suspiciously similar feature set but just enough incompatibility to require separate adapters. Microsoft folded tool-calling semantics directly into Azure AI's runtime, making their own connector framework the path of least resistance for anyone already in the Azure ecosystem. Amazon followed with Bedrock Bridges, a managed service that was "inspired by MCP" but locked to Bedrock-hosted models and demanded IAM-shaped auth flows that no other provider could replicate without selling their soul to AWS. Within a year, what was supposed to be one protocol became four — each with its own SDK, its own auth model, its own serialization quirks, and its own opinionated take on how tool descriptions should be structured.

The community tried to hold the line. The mcp-compat shim layer hit 12,000 GitHub stars in its first week, promising to translate between all four protocols at runtime. It worked for simple cases — read-a-file, query-a-database — but fell apart the moment you needed streaming responses, multi-step tool chains, or any of the advanced features that made these protocols useful in the first place. Maintaining compatibility matrices across four moving targets burned out the core contributors by mid-2025. The repo's last commit message reads simply: "I can't do this anymore. Good luck." Meanwhile, the original MCP spec itself stalled. The working group meetings became ghost towns as Anthropic's own priorities shifted toward tighter vertical integration with Claude's native tool system.

What actually replaced MCP wasn't another protocol — it was convention. Teams stopped trying to build universal connectors and instead adopted thin, bespoke HTTP wrappers around their tools, documented with OpenAPI specs and consumed directly by models that had gotten smart enough to figure out the plumbing themselves. The "protocol" became just a well-structured prompt and a JSON schema. It's less elegant, less standardized, and absolutely impossible to put in a keynote slide — but it works, it's simple, and it doesn't require buying into anyone's ecosystem. The grand vision of a universal tool protocol died not because it was a bad idea, but because the incentives of the companies building AI platforms were never aligned with true interoperability.

The Config That Time Forgot

If you're still running an MCP setup, here's what your deprecated config probably looks like. Most of these server entries will 404 by the end of the quarter: 

// ~/.config/mcp/servers.json  (deprecated)
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem"],
      "enabled": false
    },
    "postgres": {
      "command": "mcp-server-postgres",
      "args": ["--connection-string", "postgresql://localhost/mydb"],
      "enabled": false,
      "_deprecated": "Use direct HTTP bridge instead. See migration guide."
    },
    "kubernetes": {
      "command": "mcp-k8s-bridge",
      "args": ["--context", "production", "--namespace", "default"],
      "enabled": false,
      "_note": "Replaced by kubectl-ai plugin v3"
    },
    "github": {
      "command": "mcp-github",
      "args": ["--token-env", "GITHUB_TOKEN"],
      "enabled": false,
      "_sunset": "2026-03-01"
    }
  },
  "protocolVersion": "2024-11-05",
  "_warning": "This configuration format is no longer maintained."
}

Rest in peace, MCP. You deserved a better industry than the one you got.
