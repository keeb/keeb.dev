Hexo & VSCode

2022-05-28 — keeb

Setting up Hexo as a static site generator turned out to be one of those decisions that compounds in value over time. The initial draw was simplicity: Markdown files in, static HTML out, no database to babysit, no PHP runtime to patch. But the real payoff came from how naturally it fits into a developer workflow. You write posts in the same editor you write code in, you version control them with Git, and deployment is a single command away.

VSCode becomes the perfect companion here. With its built-in Markdown preview, integrated terminal, and Git support, you never have to leave the editor. I have a workspace configured with the Hexo project at the root, a couple of snippets for frontmatter templates, and a task that runs hexo server in the background for live preview. The feedback loop is tight: save a file, glance at the browser, iterate.

The theme system is where things get interesting. Hexo uses EJS templates and Stylus for CSS preprocessing, which gives you enough rope to build something genuinely custom without drowning in framework abstractions. The Cactus theme I started from was clean and minimal, but over time it has accumulated layers of personality—custom color schemes, modified layouts, little visual touches that make the site feel less like a template and more like a place.

The best blogging setup is the one that gets out of your way and lets you write. Hexo with VSCode is exactly that—invisible infrastructure that just works.
