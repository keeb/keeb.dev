Hexo & VSCode

2022-05-28 — keeb

Hexo is the static site generator used to generate this blog, along with a modified version of the cactus theme. What I like about hexo is that it is easy to install with npm install hexo-cli -g and set up with hexo init blog and start a server with hexo server and create new posts with hexo new "Post". Themes are just a simple drop into the themes directory, update the _config.yml to point to the correct theme, and you are off and running.

After more than 25 years using vim and carrying around my lightly modified .vimrc everywhere, I am starting to explore using IDEs. For this project I have been using VSCode because I appreciate the live preview of Markdown as I type and it is much better than SublimeText. One habit that has not shaken, however, is that I really like to develop on Linux and I prefer to develop remotely. This allows me to work from any internet-connected device, seamlessly, without having to copy files or worry about committing changes in progress.

My code is just one ssh command away.

VSCode takes this a step further and can effectively mount remote filesystems and load source code as if it is local, seamlessly. The only non-trivial part of this setup is that the default launch experience in VSCode does not work with Hexo, because it expects to run node on a javascript file somewhere, but no such javascript exists. This is remedied by configuring a custom launch.json that points to the hexo binary in node_modules and passes the appropriate arguments. Once configured, you can launch a hexo server instance directly from VSCode and preview changes as you make them.
