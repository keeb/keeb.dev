Hexo & VSCode

2022-09-14

I finally sat down and wired up a proper blogging workflow. The stack is simple on purpose: Hexo generates the static files, a custom Cactus theme handles the look, and the whole thing ships inside a Docker container that Nginx serves to the world. No CMS, no database, no JavaScript framework on the frontend. Just markdown files rendered into HTML, the way the web was meant to work.

The editing loop

VSCode is where all the writing happens. I keep the site repo open with the integrated terminal split below, running hexo server in watch mode so every save triggers a rebuild. The preview refreshes in a side browser tab, and the feedback loop is tight enough that I never leave the editor. A couple of workspace-level snippets handle front-matter boilerplate: type newpost, tab, fill in the title, and I am writing within seconds. The hexo-asset-link plugin resolves image paths from per-post asset folders, so dropping a screenshot into the right directory is all it takes to embed it.

The best writing setup is the one where the distance between thought and published word is as short as possible.

Deployment is a single ./scripts/deploy.sh invocation. The script pulls the latest Docker image, runs hexo generate inside the container, and drops the output into a timestamped directory. A symlink swap makes the new version live atomically -- no downtime, no half-rendered pages. If something goes wrong I can point the symlink back to the previous build in under a second. It is not Kubernetes, but for a personal site it hits the sweet spot between reliability and simplicity.

Why this matters

There is something satisfying about a deploy pipeline you can hold in your head entirely. Every piece is visible: the markdown source, the theme templates, the Nginx config, the shell script that ties them together. When the whole system fits on a single screen of documentation, you actually maintain it instead of letting it rot. That is the real payoff -- not the tech, but the fact that six months from now I will still understand how it works and will still want to write.
